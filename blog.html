Scope of var, let, and const

var

Function Scope: Variables declared with var are function-scoped. This means they are only accessible within the function in which they are defined. Hoisting: Variables declared with var are hoisted, which means they are moved to the top of their function or global scope during compilation.

let

Block Scope: Variables declared with let are block-scoped. They are accessible only within the block (a pair of curly braces) where they are defined. No Hoisting: Unlike var, variables declared with let are not hoisted, so you cannot access them before they are declared.

const

Block Scope: Like let, variables declared with const are also block-scoped. Immutable: Variables declared with const cannot be reassigned after they are initialized. However, for objects and arrays declared with const, their properties or elements can be modified.

Use Cases of null and undefined

NULL

  • Represents an intentional absence of any object value.
  • It is often used to signify that a variable, property, or value has no meaningful value.
  • It is assigned explicitly to indicate that a variable should have no value, whereas undefined typically represents unintentional absence or lack of initialization.

undefined

  • Typically represents an unintentional absence or lack of initialization of a value or variable.
  • When a variable is declared but not assigned a value, it is undefined by default.
  • It can also be explicitly assigned to indicate the absence of a value.

  • REST is an architectural style for designing networked applications.
  • REST APIs use standard HTTP methods (GET, POST, PUT, DELETE) to interact with resources identified by URLs.
  • They are stateless and use JSON or XML for data exchange, facilitating communication between different systems over the internet.
